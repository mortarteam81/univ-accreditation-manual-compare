## 3. SIMILAR_TO 엣지 품질 점검 결과 (자동 분석)

### 점수 분포 (5,097건)
- min=0.550, median=0.757, mean=0.788, max=1.000
- 1.0 만점 쌍 1,107건 — 5개 파일 간 **동일 제목의 준거가 서로 연결**된 케이스 (예: `crit_4_overview_1.5` = "교육성과" ↔ `crit_3_checkpoints_5.2` = "교육성과")
- 0.55~0.7 구간 1,768건 (임계값 근처 — "약한 후보")

### 교차 관계
- cross-cycle (3↔4): 2,582건, same-cycle: 2,515건
- cross-part (서로 다른 파일의 준거/항목 연결): **2,546건 / 50%**
  - 개요↔근거자료, 점검사항↔유의사항 등 "같은 준거 번호가 서로 다른 편람 섹션에 등장" 관계를 자동 포착
  - 대시보드에서 "같은 주제의 5개 섹션을 한 번에 보여주기" 기능 구현 가능

### 모델 품질 신호 (중요)
- cross-cycle SIMILAR_TO 중 1,950건이 기존 MAPS_TO 에 이미 있는 준거 관계를 **재발견** (준거 단위 집계)
  → Phase 1 검수 매핑 중 상당 부분을 모델이 자동으로 재현 = 모델이 도메인에 충분히 유효함
- 반면 cycle×criterion 단위로 **기존 매핑에 없는 새 후보 304쌍** 제시
  → Phase 3에서 "검토 필요" 표시로 노출할 가치 있음 (완전히 새로운 준거 연관성 후보)

## 5. Phase 3 다음 세션 작업 계획

### 대시보드 3탭 구성 (확정)
1. **메인 탭 — 매핑 탐색**
   - 좌: 3주기 준거 트리 (파트별 필터)
   - 중: 매핑 화살표 (변경 유형 색상: 유지/수정/통합/분할/이동)
   - 우: 4주기 준거 트리
   - 준거 클릭 시 SIMILAR_TO 후보 표시 (점수 0.7+만, 토글로 확장)
2. **영향도 패널** (메인 클릭 시 하단/우측 등장)
   - MAPS_TO + ITEM_MAPS_TO + SIMILAR_TO 를 통합한 "이 준거의 변경 영향 범위"
   - item 수준 텍스트 diff (Phase 1 mapping_note 활용)
3. **전체 구조 탭 — Sankey/Chord**
   - 전체 24그룹 × 5 파일의 흐름을 한눈에

### 기술 스택
- 단일 HTML + D3.js + Tailwind CDN
- 로컬 JSON fetch (file:// 에서도 동작하도록 embedded JS data 옵션도 고려)

### 설계 결정사항 (메모리에 이미 반영됨)
- 임베딩 모델/하이퍼파라미터 확정: paraphrase-multilingual-MiniLM-L12-v2 + alpha 0.7 + top-k 5 + threshold 0.55 + --all-pairs
- mapping_type 통일 체계: Phase 1 기준 적용
